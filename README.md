# פרויקט גמר – רשתות תקשורת (Computer Networks Final Project)

**מגישים:** נתנאל ציפלקוב, מתיתיהו ציפלקוב, טוני אטמה

## 📖 מבוא ומטרות הפרויקט
מטרת פרויקט זה היא להעמיק את ההבנה המעשית והעיונית של עקרונות התקשורת ברשתות מחשבים, בדגש על מודל השכבות TCP/IP, תהליך אריזת הנתונים (Encapsulation) וניתוח תעבורת רשת באמצעות Wireshark.
הפרויקט כולל תכנון ומימוש של יישום רשת מבוזר מבוסס פרוטוקול TCP המדגים תקשורת דו־כיוונית בין שרת למספר לקוחות במקביל.

הפרויקט מחולק לשני חלקים עיקריים:
1. **חלק 1:** אריזת נתונים ולכידת מנות בעזרת Wireshark.
2. **חלק 2:** פיתוח יישום צ'אט מבוסס שרת–לקוח וניתוח תעבורת הרשת שלו.

---

## 🛠️ חלק 1: אריזת נתונים ולכידת מנות
בשלב זה בוצעה הדמיה של מעבר מידע לוגי משכבת היישום ועד לשכבת הקישור הפיזית.

### תהליך העבודה:
* **יצירת נתונים:** הוכן קובץ CSV המכיל הודעות משכבת היישום עם שדות כגון מזהה הודעה, פרוטוקול ותוכן.
* **הדמיית Encapsulation:** באמצעות מחברת Jupyter, ההודעה נארזה למקטע TCP, לאחר מכן לחבילת IP ולבסוף למסגרת של שכבת הקישור.
* **ניתוח ב-Wireshark:** בוצעה לכידת תעבורה בזמן אמת ונשמרה לקובץ pcap. הניתוח אפשר להבחין בשדות כגון Source Port, Destination Port ו-Sequence Number.

---
## 💬 חלק 2: פיתוח יישום רשת (Chat App)
* **"Lord Of The Pings"**.
* היישום הוא מערכת צ'אט מבוזרת המאפשרת תקשורת דו־כיוונית בזמן אמת.

### תכונות המערכת:
* **ריבוי לקוחות:** תמיכה במספר לקוחות במקביל באמצעות ספריית threading ב-Python.
* **סוגי הודעות:** אפשרות לשליחת הודעות פומביות לכלל המשתמשים או הודעות פרטיות ללקוח יעד.
* **גילוי שרת אוטומטי (Discovery):** שימוש בפרוטוקול UDP לאיתור השרת ברשת.
* **ממשק גרפי (GUI):** מימוש ממשק גרפי אינטראקטיבי הן לשרת והן ללקוח.
* **חוסן:** טיפול במצבי שגיאה וניתוקים בלתי צפויים של לקוחות.

### טכנולוגיות:
* **שפת פיתוח:** Python.
* **ספריות מרכזיות:** socket, threading, customtkinter.

---

## 🚀 הוראות התקנה והרצה
1. וודאו שמותקנת סביבת **Python 3**.
2. התקינו את הספריות הנדרשות: `pip install customtkinter`.
3. הפעילו תחילה את קובץ השרת.
4. הפעילו אחד או יותר מקבצי הלקוח והתחברו לשרת.

## 🤖 שימוש בבינה מלאכותית
במהלך הפרויקט נעשה שימוש בכלי AI לצורכי ייעוץ, בדיקת תחביר (Syntax) ו-Debugging. השימוש סייע בזיהוי שגיאות ושיפור קריאות הקוד.

---

## 📊 ניתוח תעבורת היישום
בעת הרצת היישום, נלכדה התעבורה ב-Wireshark ונצפו השלבים הבאים:
* **TCP 3-Way Handshake:** שלבי ה-SYN, SYN-ACK, ACK בפתיחת החיבור.
* **העברת נתונים:** העברת נתונים דו-כיוונית בין השרת ללקוחות.
